配置一个 适应 Ubuntu 20.04 和 RTX 3050 显卡的PyTorch 环境





第一步：创建专属环境

为了不干扰系统，我们建立一个名为 clip\_env 的独立房间，并指定 Python 3.9（这个版本对 CLIP 和现代库支持最好）

在终端输入：

conda create -n clip\_env python=3.9 -y

**创建完成后，激活它（注意：以后每次要跑 CLIP 代码前，都要输这行命令）：**

**conda activate clip\_env**



Conda 的默认设置里有一个“隐形”的机制：如果你不明确修改 default\_channels（默认频道），它依然会偷偷尝试连接那个需要同意条款的官方源 (repo.anaconda.com)。



我们需要用一个更彻底的配置来覆盖它。



请按照以下步骤，彻底切断与官方源的联系，全部走清华镜像（这不仅能解决报错，还能让以后下载更快）。



1.使用“终极版”配置文件 (覆盖旧设置)

请直接复制下面这一整块代码（从 cat 开始到 EOF 结束），粘贴到终端里并回车。



这会告诉 Conda：“默认频道就是清华源，别去连官方的了”。

cat > ~/.condarc <<EOF

channels:

  - defaults

show\_channel\_urls: true

default\_channels:

  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main

  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/r

  - https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/msys2

custom\_channels:

  conda-forge: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

  msys2: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

  bioconda: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

  menpo: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

  pytorch: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

  pytorch-lts: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

  simpleitk: https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud

EOF

2.再次清理缓存

conda clean -i -y

3.重新创建环境

conda create -n clip\_env python=3.9 -y

4.激活

conda activate clip\_env





第二步：安装pytorch（GPU版）

conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia -y

Conda 会自动为你下载匹配你显卡的 CUDA 工具包，你不需要自己在系统里装 CUDA。





第三步：安装 Git 和 CLIP 模型

新装的 Ubuntu 可能还没有 Git，我们需要先装上。

1.安装系统工具 Git：

sudo apt update

sudo apt install git -y

2.安装 CLIP 及其依赖： 根据你之前上传的文档要求，我们需要安装 ftfy, regex, tqdm 以及 CLIP 本体。

pip install ftfy regex tqdm

pip install git+https://github.com/openai/CLIP.git
